# Task ID: 6
# Title: [1차] 집중 타이머 기능 구현
# Status: pending
# Dependencies: 5
# Priority: medium
# Description: 사용자가 목표에 집중할 수 있도록 돕는 뽀모도로 및 커스텀 타이머 기능 구축
# Details:
뽀모도로 프리셋(25/5분 작업/휴식 주기) 및 사용자 지정 기간 옵션이 있는 타이머 컴포넌트를 만듭니다. 타이머 컨트롤(시작, 일시 중지, 재설정, 건너뛰기)을 구현합니다. Supabase에서 타이머 세션 데이터 모델을 구축합니다. 타이머 세션을 특정 목표와 연결하는 기능을 추가합니다. 세션 기록 및 통계를 구현합니다. 타이머 이벤트에 대한 시각적 및 오디오 알림을 만듭니다.

# Test Strategy:
다양한 세션 기간에 걸쳐 타이머 정확도를 테스트합니다. 사용자가 다른 페이지로 이동해도 타이머 상태가 유지되는지 확인합니다. 타이머 세션과 목표 간의 연결을 테스트합니다. 타이머 통계가 올바르게 계산되고 저장되는지 검증합니다.

# Subtasks:
## 1. 타이머 UI 컴포넌트 개발 [pending]
### Dependencies: None
### Description: 뽀모도로 및 커스텀 타이머를 위한 UI 컴포넌트를 개발합니다.
### Details:
1. 타이머 디스플레이 컴포넌트 구현 (시간, 분, 초 표시)
2. 타이머 컨트롤 버튼 구현 (시작, 일시 중지, 재설정, 건너뛰기)
3. 뽀모도로 프리셋 선택 UI 구현 (25/5, 50/10 등)
4. 커스텀 타이머 설정 UI 구현 (사용자 지정 작업/휴식 시간)
5. 현재 타이머 상태 표시 (작업 중, 휴식 중)
6. 반응형 디자인 적용
7. 테스트: 타이머 UI가 모든 상태에서 올바르게 표시되는지 확인

## 2. 타이머 로직 및 상태 관리 구현 [pending]
### Dependencies: 6.1
### Description: 타이머 기능의 핵심 로직과 상태 관리를 구현합니다.
### Details:
1. 타이머 카운트다운 로직 구현
2. 작업/휴식 주기 전환 로직 구현
3. 타이머 상태 관리 (실행 중, 일시 중지, 중지)
4. 브라우저 탭 전환 시 타이머 정확성 유지 로직 구현
5. 로컬 스토리지를 사용한 타이머 상태 유지 구현
6. 테스트: 다양한 시나리오에서 타이머 로직이 정확하게 작동하는지 확인

## 3. 타이머 세션 데이터 모델 및 API 구현 [pending]
### Dependencies: 6.2
### Description: 타이머 세션을 저장하고 관리하기 위한 데이터 모델과 API를 구현합니다.
### Details:
1. Supabase에 타이머 세션 테이블 생성 (user_id, goal_id, start_time, end_time, duration, type 등 포함)
2. 세션 시작 API 엔드포인트 구현
3. 세션 종료/업데이트 API 엔드포인트 구현
4. 세션 조회 API 엔드포인트 구현
5. 목표별 세션 통계 API 엔드포인트 구현
6. 테스트: API 엔드포인트가 올바르게 작동하는지 확인

## 4. 목표 연결 기능 구현 [pending]
### Dependencies: 6.2, 6.3
### Description: 타이머 세션을 특정 목표와 연결하는 기능을 구현합니다.
### Details:
1. 타이머 시작 전 목표 선택 UI 구현
2. 최근/자주 사용한 목표 빠른 선택 기능 구현
3. 선택한 목표 정보 표시 UI 구현
4. 세션 중 목표 변경 기능 구현 (선택적)
5. 목표 없이 타이머 사용 옵션 구현
6. 테스트: 목표 선택 및 연결 기능이 올바르게 작동하는지 확인

4. 작업/휴식 전환 시 알림 구현
5. 알림 설정 UI 구현 (소리 켜기/끄기, 볼륨 등)
6. 테스트: 다양한 브라우저에서 알림이 올바르게 작동하는지 확인
